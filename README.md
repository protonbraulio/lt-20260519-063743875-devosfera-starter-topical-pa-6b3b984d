## Commands

- `pnpm dev`
- `pnpm build`
- `pnpm preview`

## Blog content

- Posts live in `src/content/blog`
- Frontmatter supports `title`, `description`, `publishedAt`, optional `updatedAt`, optional `draft`, optional `tags`, optional `category`, and optional `image`
- A post is public only when `draft` is not `true` and `publishedAt` is in the past

## Scheduled publishing

- Scheduled publishing config lives in `staxio/publishing.config.json`
- Prepared posts use `status: scheduled`, `draft: true`, and `publishAt`
- `.github/workflows/publish-scheduled-posts.yml` only flips frontmatter and commits due posts
- The scheduler does not install dependencies, run Astro builds, or call LLM APIs
- Push-triggered deploys publish the site after the scheduler commit lands

## Robots and sitemap

- `robots.txt` is generated from `src/pages/robots.txt.ts`
- Submit `/sitemap-index.xml` to crawlers and search consoles

## RSS feed

- `src/pages/rss.xml.ts` generates a feed from published blog posts
- Draft and future-dated posts are excluded from the feed

## Basic pages

- The starter includes `/about`, `/contact`, `/privacy-policy`, `/terms`, and `/cookie-policy`
- These pages are generated with template-aware copy and are linked from the homepage footer
- Review and customize the content before production use if you need site-specific legal or business details
